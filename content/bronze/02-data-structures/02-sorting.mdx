Sorting is one of the most fundamental operations in competitive programming. A huge number of problems become trivial once the data is sorted.

## Using STL Sort

In competitive programming, you almost never implement your own sorting algorithm. C++ provides `sort()` from the `<algorithm>` header:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    vector<int> v = {5, 2, 8, 1, 9, 3};
    
    sort(v.begin(), v.end());       // ascending: {1, 2, 3, 5, 8, 9}
    sort(v.rbegin(), v.rend());     // descending: {9, 8, 5, 3, 2, 1}
    
    return 0;
}
```

<Info>
`sort()` uses **IntroSort** (a hybrid of quicksort, heapsort, and insertion sort) and runs in $O(N \log N)$ — which is optimal for comparison-based sorting.
</Info>

## Sorting Arrays

```cpp
int arr[6] = {5, 2, 8, 1, 9, 3};
sort(arr, arr + 6); // sort all 6 elements
```

## Custom Comparators

When you need to sort by something other than the default order, write a custom comparator:

### Using a Function

```cpp
bool compare(int a, int b) {
    return a > b; // sort descending
}

sort(v.begin(), v.end(), compare);
```

### Using a Lambda

```cpp
sort(v.begin(), v.end(), [](int a, int b) {
    return a > b; // sort descending
});
```

### Sorting Pairs

By default, pairs sort by first element, then by second:

```cpp
vector<pair<int, int>> v = {{3, 1}, {1, 5}, {3, 2}, {1, 3}};
sort(v.begin(), v.end());
// Result: {1,3}, {1,5}, {3,1}, {3,2}
```

To sort by second element:

```cpp
sort(v.begin(), v.end(), [](auto &a, auto &b) {
    return a.second < b.second;
});
```

## Sorting Structs

For more complex data, define a struct and a comparator:

```cpp
struct Student {
    string name;
    int score;
};

int main() {
    vector<Student> students = {
        {"Alice", 85},
        {"Bob", 92},
        {"Charlie", 78}
    };
    
    // Sort by score (descending), then by name (ascending)
    sort(students.begin(), students.end(), [](const Student &a, const Student &b) {
        if (a.score != b.score) return a.score > b.score;
        return a.name < b.name;
    });
    
    for (auto &s : students) {
        cout << s.name << " " << s.score << "\n";
    }
}
```

<Warning>
Your comparator must define a **strict weak ordering**. It must return `false` when comparing equal elements: `compare(a, a)` must be `false`. Using `<=` instead of `<` will cause undefined behavior and potential crashes!
</Warning>

## Common Sorting Patterns

### Coordinate Compression

When values are large but you only care about relative order:

```cpp
vector<int> v = {1000000, 3, 500000, 1};
vector<int> sorted_v = v;
sort(sorted_v.begin(), sorted_v.end());
sorted_v.erase(unique(sorted_v.begin(), sorted_v.end()), sorted_v.end());

for (int &x : v) {
    x = lower_bound(sorted_v.begin(), sorted_v.end(), x) - sorted_v.begin();
}
// v is now {3, 1, 2, 0}
```

### Sorting by Index

Sometimes you want to sort indices by their corresponding values:

```cpp
vector<int> v = {30, 10, 50, 20};
vector<int> idx(v.size());
iota(idx.begin(), idx.end(), 0); // {0, 1, 2, 3}

sort(idx.begin(), idx.end(), [&](int a, int b) {
    return v[a] < v[b];
});
// idx = {1, 3, 0, 2} (indices sorted by values)
```

<Spoiler title="Stable Sort">
`sort()` is not guaranteed to preserve the relative order of equal elements. If you need stability (equal elements maintain their original order), use `stable_sort()`:

```cpp
stable_sort(v.begin(), v.end(), compare);
```

This is useful when you sort by one criterion and want to preserve a previous sort order for ties.
</Spoiler>
